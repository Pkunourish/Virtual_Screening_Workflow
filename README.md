# Virtual_Screening_Workflow

a Python-based workflow for data processing after virtual screening. Vina and Glide results as example. Codes are provided with .ipynb and .py format. Detailed footnotes are provided within .py code files.

Carried out in following order: vina/glide → merge → RMSD → PharmFit → SASA → pains → cluster

Result files are not provided. You may need to extract the original score files and binding poses with your own codes.
